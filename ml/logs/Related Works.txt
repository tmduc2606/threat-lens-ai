I. Related Works:
1. Discerning Reliable Cyber Threat Indicators for Timely Cyber Threat Intelligence (2023) - compares IoCs against threat-intelligence services including AlienVault OTX, MalwareBazaar, and VirusTotal, and evaluates accuracy, latency, and exclusiveness of indicators. This is a strong predecessor for the OTX pulse part of your dataset.
2. A Risk Manager for Intrusion Tolerant Systems: Enhancing HAL 9000 with New Scoring and Data Sources (2025) - combines EPSS, CVSS, and AlienVault OTX pulses to rescore CVEs, explicitly treating pulse occurrences as evidence of active exploitation. This is one of the closest “fusion” works for your CVE + threat-intel side.
3. VulZoo: A Comprehensive Vulnerability Intelligence Dataset (2024) - a vulnerability-intelligence dataset built around NVD, CISA KEV, CWE, CAPEC, ATT&CK, D3FEND, and AttackerKB, with utility scripts for synchronization and cleaning. This is a very strong related work for your CVE vulnerabilities file.
4. Efficacy of EPSS in High Severity CVEs found in CISA KEV (2024) - studies whether EPSS can predict exploitation for vulnerabilities already known to be exploited, using the CISA KEV catalog as the reference set. Good support for vulnerability-prioritization framing.
5. Vulnerability Management Chaining: An Integrated Framework for Efficient Cybersecurity Risk Prioritization (2025) - integrates KEV + EPSS + CVSS into a decision-tree framework and validates it on 28,377 vulnerabilities. This is a strong “successor” style work if you want to argue for multi-source vulnerability prioritization.
6. From Past to Present: A Survey of Malicious URL Detection Techniques, Datasets and Code Repositories (2025) - surveys malicious URL detection from blacklists through Transformers, GNNs, and LLMs, and organizes the field by modality. This is a good umbrella citation for the malicious domain/URL branch of your dataset.
7. DomURLs_BERT: Pre-trained BERT-based Model for Malicious Domains and URLs Detection and Classification (2024) - pretrains a BERT-style encoder on multilingual URLs, domain names, and DGA data, then evaluates phishing, malware, DGA, and DNS-tunneling tasks. This is one of the clearest deep-learning predecessors for your malicious domain file.
8. Combining NLP and MLP for Enhanced Detection of Suspicious Domains (2024) - builds a real-life domain pipeline, enriches domain records with similarity and semantic/numeric features, and reports F1 84.86% and accuracy 84.95% on the SecReg dataset. This is especially relevant because your domain file also contains registration-time / WHOIS-like features.
9. Phishing Detection Leveraging Machine Learning and External Features (2022) - explicitly uses external features such as IP reputation, domain reputation, and WHOIS registration information, while noting the cost of maintaining those external data sources. That maps very well to your malicious-domain schema.
10. Detecting Malicious Domains Using Statistical Features (2022) - uses malicious domain lists such as DNS-BH and PhishTank and relies on DNS/IDN statistical features for classification. This is a classic baseline style related work for malicious-domain detection.
11. AI-Driven Fast and Early Detection of IoT Botnet Threats (2024) - analyzes IoT network traffic with semi-supervised learning to detect stealth C2 communications, reporting strong results on IoT-23. This is relevant to your malicious IP file because it addresses botnet / command-and-control behavior at the network level.
12. Comprehensive Botnet Detection by Mitigating Adversarial Attacks (2024) and ML Defender (aRGus NDR) (2026) - both are flow-based botnet/NIDS works; the former focuses on adversarial robustness, while the latter is an open-source embedded ML NIDS for botnet and anomalous traffic detection with real deployment emphasis. These are useful if you want to connect malicious IPs to network-behavior detection rather than pure reputation feeds.
