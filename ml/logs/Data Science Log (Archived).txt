Dataset Introduction
The dataset combines four complementary threat perspectives: OTX pulses for cyber threat intelligence narratives, CVE records for exploited vulnerabilities, malicious domains for phishing and C2 infrastructure, and malicious IPs for scanning, botnet, and exfiltration activity. 
Name: Cybersecurity Threat Intelligence Dataset 2026
Sources: Various websites (AlienVault OTX, CISA, and NVD)
Preliminaries: Already performed data cleaning procedures
What’s Inside:
4,312 total records across 4 files:
OTX Threat Intelligence - 2,365 records | Real threat pulses from AlienVault's global researcher network. Includes malware families, targeted countries, attack techniques, and adversary groups. Good for building threat classification models or studying APT behavior.
CVE Vulnerabilities - 1,585 records | Actively exploited vulnerabilities from 2024 to 2026, sourced from CISA's KEV catalog and NVD. Each entry has CVSS scores, affected vendor/product, and patch status. Useful for vulnerability prioritization research.
Malicious Domains - 162 record | Domains flagged for phishing, C2 activity, and malware distribution. Includes registrar info, country, and first/last seen dates.
Malicious IPs - 200 records | IP addresses reported for scanning, brute force, botnet activity, and exfiltration. Includes ASN, country, and reputation scores.

Related Works
Discerning Reliable Cyber Threat Indicators for Timely Cyber Threat Intelligence (2023) - compares IoCs against threat-intelligence services including AlienVault OTX, MalwareBazaar, and VirusTotal, and evaluates accuracy, latency, and exclusiveness of indicators. This is a strong predecessor for the OTX pulse part of your dataset.
A Risk Manager for Intrusion Tolerant Systems: Enhancing HAL 9000 with New Scoring and Data Sources (2025) - combines EPSS, CVSS, and AlienVault OTX pulses to rescore CVEs, explicitly treating pulse occurrences as evidence of active exploitation. This is one of the closest “fusion” works for your CVE + threat-intel side.
VulZoo: A Comprehensive Vulnerability Intelligence Dataset (2024) - a vulnerability-intelligence dataset built around NVD, CISA KEV, CWE, CAPEC, ATT&CK, D3FEND, and AttackerKB, with utility scripts for synchronization and cleaning. This is a very strong related work for your CVE vulnerabilities file.
Efficacy of EPSS in High Severity CVEs found in CISA KEV (2024) - studies whether EPSS can predict exploitation for vulnerabilities already known to be exploited, using the CISA KEV catalog as the reference set. Good support for vulnerability-prioritization framing.
Vulnerability Management Chaining: An Integrated Framework for Efficient Cybersecurity Risk Prioritization (2025) - integrates KEV + EPSS + CVSS into a decision-tree framework and validates it on 28,377 vulnerabilities. This is a strong “successor” style work if you want to argue for multi-source vulnerability prioritization.
From Past to Present: A Survey of Malicious URL Detection Techniques, Datasets and Code Repositories (2025) - surveys malicious URL detection from blacklists through Transformers, GNNs, and LLMs, and organizes the field by modality. This is a good umbrella citation for the malicious domain/URL branch of your dataset.
DomURLs_BERT: Pre-trained BERT-based Model for Malicious Domains and URLs Detection and Classification (2024) - pretrains a BERT-style encoder on multilingual URLs, domain names, and DGA data, then evaluates phishing, malware, DGA, and DNS-tunneling tasks. This is one of the clearest deep-learning predecessors for your malicious domain file.
Combining NLP and MLP for Enhanced Detection of Suspicious Domains (2024) - builds a real-life domain pipeline, enriches domain records with similarity and semantic/numeric features, and reports F1 84.86% and accuracy 84.95% on the SecReg dataset. This is especially relevant because your domain file also contains registration-time / WHOIS-like features.
Phishing Detection Leveraging Machine Learning and External Features (2022) - explicitly uses external features such as IP reputation, domain reputation, and WHOIS registration information, while noting the cost of maintaining those external data sources. That maps very well to your malicious-domain schema.
Detecting Malicious Domains Using Statistical Features (2022) - uses malicious domain lists such as DNS-BH and PhishTank and relies on DNS/IDN statistical features for classification. This is a classic baseline style related work for malicious-domain detection.
AI-Driven Fast and Early Detection of IoT Botnet Threats (2024) - analyzes IoT network traffic with semi-supervised learning to detect stealth C2 communications, reporting strong results on IoT-23. This is relevant to your malicious IP file because it addresses botnet / command-and-control behavior at the network level.
Comprehensive Botnet Detection by Mitigating Adversarial Attacks (2024) and ML Defender (aRGus NDR) (2026) - both are flow-based botnet/NIDS works; the former focuses on adversarial robustness, while the latter is an open-source embedded ML NIDS for botnet and anomalous traffic detection with real deployment emphasis. These are useful if you want to connect malicious IPs to network-behavior detection rather than pure reputation feeds.
Proposal
Problem Framing (5%)
This project adheres to the problem of how to build a cybersecurity intelligence system that can learn from various real-life threat data sources and support automated threat categorization, vulnerability prioritization, and malicious infrastructure detection.
The central challenge is not only classification, but also representation learning across heterogeneous cybersecurity data.
Accordingly, the project aims to answer the following question: Can a machine learning pipeline effectively analyze various cybersecurity intelligence sources and produce useful threat insights to enhance security protection?
To resolve this, the study will compare traditional machine learning methods with a Transformer-based deep learning component, evaluate their performance across the four datasets, and examine how feature engineering and model choice shapes the cybersecurers.
In a nutshell, the project focuses on:
Threat classification from OTX pulses,
Exploitation-risk prioritization from CVE records
Malicious domain detection
Malicious IP classification
Practical evaluation for a deployable cybersecurity analytics prototype.
The emphasis is on data-driven threat intelligence modeling using cleaned, labeled, and real-world cybersecurity datasets.

Preprocessing, Feature Engineering (25 - 30%)
This stage prepares the four cybersecurity datasets for modeling by converting raw cleaned records into model-ready features:
Schema harmonization: standardize column names, check data types, and align the structure of all four CSV files.
Datetime conversion: convert time-related fields such as created, modified, due dates, and analysis dates into proper datetime format.
Text preprocessing for OTX: clean threat text fields like titles, descriptions, tags, malware families, and ATT&CK IDs using lowercasing, punctuation cleanup, tokenization, and optional stopword removal.
Categorical encoding: transform fields such as vendor, product, registrar, country, ASN, threat category, and severity using label encoding, one-hot encoding, or frequency-based encoding depending on the model.
Numerical scaling and engineering: normalize numeric fields such as reputation scores, counts, subscribers, votes, and report totals; also derive useful ratios where needed.
Security-specific feature engineering: create domain-relevant indicators such as domain length patterns, hyphen/number presence, WHOIS-based age features, TOR-node flags, exploitation-related CVE fields, and ATT&CK or malware-family references.
Model-specific representation: use TF-IDF or character n-grams for classical ML text pipelines, and raw or lightly cleaned text for the Transformer-based OTX model.
Missing data and duplicates handling: impute or fill missing values where necessary, replace empty text with neutral placeholders, and remove or consolidate duplicates.
Final model-ready outputs: produce separate feature sets for OTX, CVE, malicious domains, and malicious IPs, each matched to its own modeling strategy.
Modeling (25 - 30%)
OTX → TF-IDF / char n-grams + Logistic Regression or Linear SVM | Deep Learning Approach: Transformer-based CTI text model (BERT / Transformer fine-tuning)
CVE → Random Forest / XGBoost-style boosting, calibration or ranking metrics
Domains/IPs → boosted trees on engineered features, with char n-grams as an optional add-on for domains
Evaluation (15%)
OTX threat intelligence task: Classification metrics such as accuracy, precision, recall, F1-score, and macro-averaged F1, especially when threat categories are imbalanced. Multi-label metrics such as micro-F1, macro-F1, Hamming loss, and subset accuracy may be used for multi-label outputs such as malware families, ATT&CK techniques, or threat tags.
CVE prioritization task: In addition to standard classification metrics, ROC-AUC, PR-AUC, precision@k, recall@k, and calibration metrics are useful for measuring how well the model distinguishes high-risk vulnerabilities from lower-priority ones.
Malicious domain and malicious IP tasks: Metrics such as precision, recall, F1-score, ROC-AUC, and confusion matrices can reveal how effectively the model identifies malicious infrastructure while limiting false alarms.
Finally, to ensure a reliable comparison across methods, the evaluation should use a consistent validation strategy such as train-validation-test splitting or stratified cross-validation where feasible.

def collapse_attack_id(attack_id: str) -> str:
    return attack_id.split(".")[0]

def preprocess_otx(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()

    # Text normalization
    text_cols = ["Title", "Description", "Tags", "Malware_Families", "Industries", "Countries"]
    for col in text_cols:
        if col in df.columns:
            df[col] = df[col].fillna("").map(normalize_text)

    # Numeric cleanup
    df["Indicators_Count"] = (pd.to_numeric(df.get("Indicators_Count"), errors="coerce").fillna(0).clip(0, 10000))
    df["Subscribers"] = (pd.to_numeric(df.get("Subscribers"), errors="coerce").fillna(0).clip(0, 100000))

    # Log transform helps small datasets
    df["log_indicators"] = np.log1p(df["Indicators_Count"])
    df["log_subscribers"] = np.log1p(df["Subscribers"])

    # Time feature
    df["pulse_age_days"] = ((df["Modified"] - df["Created"]).dt.days.fillna(0).clip(lower=0))
    df["log_pulse_age"] = np.log1p(df["pulse_age_days"])

    # ATT&CK multi-label
    df["Attack_List"] = df["Attack_IDs"].apply(split_multi_value)
    df["Attack_List"] = df["Attack_List"].apply(lambda x: [collapse_attack_id(i) for i in x])

    # Filter rare labels
    counter = Counter(l for row in df["Attack_List"] for l in row)

    valid_labels = {k for k, v in counter.items() if v >= MIN_ATTACK_FREQ}

    df["Attack_List"] = df["Attack_List"].apply(lambda x: [i for i in x if i in valid_labels])

    # Remove empty rows
    df = df[df["Attack_List"].map(len) > 0].reset_index(drop=True)

    df["combined_text"] = (
        df["Title"] + " " +
        df["Description"] + " " +
        df["Tags"] + " " +
        df["Malware_Families"] + " " +
        df["Industries"] + " " +
        df["Countries"]
    ).str.replace(r"\s+", " ", regex=True).str.strip()

    df = df.drop_duplicates(subset=["combined_text", "Attack_List"])

    return df

# CVE Pipeline
def preprocess_cve(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()

    # Text cleanup
    text_cols = ["vulnerabilityName", "shortDescription", "requiredAction"]

    for col in text_cols:
        if col in df.columns:
            df[col] = df[col].fillna("").map(normalize_text)

    # Categorical cleanup
    cat_cols = ["vendorProject", "product", "cwes"]

    for col in cat_cols:
        if col in df.columns:
            df[col] = df[col].fillna("").astype(str).str.lower().str.strip()

    # Datetime
    df["dateAdded"] = pd.to_datetime(df["dateAdded"], errors="coerce")
    df["dueDate"] = pd.to_datetime(df["dueDate"], errors="coerce")

    df["days_to_due"] = (df["dueDate"] - df["dateAdded"]).dt.days
    df["days_to_due"] = df["days_to_due"].clip(-1, 3650)

    df["log_days_to_due"] = np.log1p(df["days_to_due"].clip(lower=0))

    # Text field (Lower Noise Design)
    df["risk_text"] = (df["vulnerabilityName"] + " " + df["shortDescription"] + " " + df["cwes"]).str.strip()

    # Structured meta field
    df["meta_text"] = (df["vendorProject"] + " " + df["product"]).str.strip()

    return df

# Domain Pipeline
def preprocess_domains(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()

    # Numeric
    numeric_cols = [
        "Domain_Length", "Reputation", "Malicious_Votes",
        "Suspicious_Votes", "Harmless_Votes", "Undetected_Votes",
        "Total_Engines"
    ]
    for col in numeric_cols:
        if col in df.columns:
            df[col] = pd.to_numeric(df[col], errors="coerce").fillna(0)

    # Date fix
    df["Creation_Date"] = pd.to_datetime(df["Creation_Date"], errors="coerce")
    df["Last_Analysis_Date"] = pd.to_datetime(df["Last_Analysis_Date"], errors="coerce")

    df["domain_age_days"] = (
        df["Last_Analysis_Date"] - df["Creation_Date"]
    ).dt.days.fillna(0).clip(lower=0)

    df["log_domain_age"] = np.log1p(df["domain_age_days"])

    # Engineered binary flags
    df["Has_Numbers"] = df["Domain"].str.contains(r"\d", na=False).astype(int)
    df["Has_Hyphen"] = df["Domain"].str.contains(r"-", na=False).astype(int)

    # Ratios
    df["malicious_ratio"] = safe_ratio(df["Malicious_Votes"], df["Total_Engines"])
    df["suspicious_ratio"] = safe_ratio(df["Suspicious_Votes"], df["Total_Engines"])

    # Log counts
    df["log_malicious"] = np.log1p(df["Malicious_Votes"])
    df["log_suspicious"] = np.log1p(df["Suspicious_Votes"])

    # Categorical cleaning
    df["TLD"] = df["TLD"].str.lower().fillna("unknown")

    return df

    numeric_cols = [
        "Malicious_Votes", "Suspicious_Votes",
        "Harmless_Votes", "Undetected_Votes",
        "Total_Reports", "Reputation_Score", "Times_Submitted"
    ]

    for col in numeric_cols:
        df[col] = pd.to_numeric(df[col], errors="coerce").fillna(0)

    # ASN as categorical
    df["ASN"] = df["ASN"].astype(str).fillna("unknown")

    # Ratios
    df["malicious_ratio"] = safe_ratio(df["Malicious_Votes"], df["Total_Reports"])
    df["suspicious_ratio"] = safe_ratio(df["Suspicious_Votes"], df["Total_Reports"])

    # Log features
    df["log_malicious"] = np.log1p(df["Malicious_Votes"])
    df["log_suspicious"] = np.log1p(df["Suspicious_Votes"])
    df["log_harmless"] = np.log1p(df["Harmless_Votes"])
    df["log_undetected"] = np.log1p(df["Undetected_Votes"])

    # Safer reputation scaling
    min_val = df["Reputation_Score"].min()
    max_val = df["Reputation_Score"].max()

    df["reputation_score_scaled"] = (
        (df["Reputation_Score"] - min_val) /
        (max_val - min_val + 1e-9)
    )

    # TOR flag robustness
    df["tor_flag"] = df["TOR_Node"].astype(str).str.lower().isin(
        ["1", "true", "yes"]
    ).astype(int)

    # IP structure feature
    df["ip_first_octet"] = df["IP"].astype(str).str.split(".").str[0]

    return df
